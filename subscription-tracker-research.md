# Ресерч: Сервис трекинга подписок и регулярных оплат

## 1. Анализ существующих готовых решений

### 1.1 Bobby (iOS/macOS) — ⭐ Лучший кандидат из готовых

- **Платформы:** iOS, macOS (Apple Silicon), visionOS
- **Русский язык:** ✅ Да (официально поддерживается)
- **Цена:** Бесплатно (до 5 подписок), All-in-one Pack ~$2–5
- **Функционал:** Ручной ввод подписок, напоминания перед оплатой, категории, iCloud Sync, конвертация валют, статистика расходов (еженедельно/ежемесячно/годовой итог)
- **Что нет:** Нет web-версии, нет явной отметки "оплачено", нет Android, нет семейного шаринга данных

**Вердикт:** Покрывает ~70% потребностей. Главные минусы — нет web-доступа, нет ручной отметки "оплачено" и нет полноценного семейного режима.

### 1.2 Notion (Шаблон подписок)

- **Платформы:** Web, iOS, macOS, Android
- **Русский язык:** ✅ Интерфейс на русском
- **Цена:** Бесплатно (для личного использования)
- **Функционал:** База данных подписок, категории, статусы, формулы для расчёта стоимости, дашборд, напоминания (через Notion Reminders)
- **Что нет:** Напоминания ограничены (нет push-уведомлений на конкретную дату оплаты, только reminder на странице), нет нативных уведомлений как у приложений

**Вердикт:** Хорошая база, но слабые напоминания. Придётся каждый раз заходить в Notion, чтобы видеть статус. Подходит если не критичны push-уведомления.

### 1.3 Google Sheets + Apps Script + Google Calendar

- **Платформы:** Web, iOS (через Google Sheets/Calendar app), macOS
- **Русский язык:** ✅ Полностью
- **Цена:** Бесплатно
- **Функционал:** Полная кастомизация структуры данных, автоматические email-напоминания через Apps Script, интеграция с Google Calendar для push-уведомлений, совместный доступ для семьи
- **Что нет:** Нет красивого UI, ввод через таблицу, требует начальной настройки скриптов

**Вердикт:** Максимальная гибкость при нулевой стоимости. Напоминания через Calendar работают на всех платформах. Но UX — спартанский.

### 1.4 Rocket Money, Monarch Money и аналоги

- **Проблема:** Привязка к банковским счетам (работают в основном в США), платные подписки ($7–14/мес), избыточный функционал, нет русского языка
- **Вердикт:** Не подходят для данного кейса.

## 3. Предлагаемые варианты реализации

### Вариант A: Google Sheets + Apps Script + Google Calendar (рекомендуемый)

**Суть:** Google Sheet как основная БД, Apps Script для автоматизации, Google Calendar для напоминаний.

**Архитектура:**
- **Google Sheet** — основная таблица с колонками: Название, Категория, Сумма, Валюта, Период (месяц/год/квартал), Дата следующей оплаты, Статус (Оплачено/Ожидает), Уведомления (вкл/выкл), Дата последней оплаты, Примечания
- **Apps Script** — скрипт, запускаемый по триггеру (ежедневно):
  - Проверяет приближающиеся оплаты
  - Создаёт/обновляет события в Google Calendar с напоминаниями
  - Отправляет email-уведомления (опционально)
  - Автоматически пересчитывает следующую дату оплаты после отметки "Оплачено"
- **Google Calendar** — отдельный календарь "Подписки" с push-уведомлениями на всех устройствах
- **Опционально:** Простая HTML-страница (Apps Script Web App) как dashboard для удобного просмотра

**Плюсы:**
- Бесплатно, нулевые затраты на хостинг
- Push-уведомления через Calendar работают на iPhone, Mac, Web
- Семейный доступ через расшаривание Sheet + Calendar
- Вся экосистема Google, знакомый интерфейс
- Можно расширить до Web App с красивым UI через Apps Script Web App

**Минусы:**
- UX ввода через таблицу (не очень удобно с телефона)
- Требует начальной настройки (~2–4 часа)
- Ограничения Apps Script (квоты на выполнение)

**Сложность реализации:** ★★☆☆☆ (Средне-низкая)

### Вариант B: Самописное Web-приложение (PWA) + Supabase

**Суть:** Fullstack PWA-приложение с бэкендом на Supabase (PostgreSQL + Auth + Cron).

**Технологический стек:**
- **Frontend:** Next.js (React) или Nuxt.js (Vue) — SSR/SSG
- **UI:** Tailwind CSS + shadcn/ui или Vuetify
- **Backend/DB:** Supabase (PostgreSQL, Auth, Edge Functions, Cron Jobs)
- **Уведомления:** Web Push API (PWA), Email через Resend/SendGrid, Telegram Bot (опционально)
- **Деплой:** Vercel (фронт) + Supabase (бэк) — всё бесплатный tier
- **i18n:** next-intl или vue-i18n для русской локализации

**Архитектура:**
```
┌──────────────────┐     ┌──────────────────┐
│   Next.js PWA    │────▶│    Supabase      │
│   (Vercel)       │     │  - PostgreSQL    │
│                  │     │  - Auth          │
│  - Dashboard     │     │  - Edge Functions│
│  - CRUD подписок │     │  - Cron (reminders)│
│  - Статистика    │     └──────────────────┘
│  - Push уведомл. │              │
└──────────────────┘              ▼
                          ┌──────────────────┐
                          │  Notifications   │
                          │  - Web Push      │
                          │  - Email         │
                          │  - Telegram Bot  │
                          └──────────────────┘
```

**Плюсы:**
- Полный контроль над UX/UI
- Работает как нативное приложение (PWA — Add to Home Screen)
- Push-уведомления на всех платформах
- Полная статистика, графики, фильтры
- Масштабируемость
- Бесплатный tier Supabase + Vercel достаточен для семьи

**Минусы:**
- Значительная разработка (~40–80 часов)
- Нужно поддерживать инфраструктуру
- PWA push на iOS имеет ограничения (с iOS 16.4+ работает, но не идеально)

**Сложность реализации:** ★★★★☆ (Высокая)

### Вариант C: Notion + Автоматизация через n8n/Make + Apple Reminders/Calendar

**Суть:** Notion как основная БД + внешний автоматизатор для напоминаний.

**Архитектура:**
- **Notion** — база данных подписок со всеми полями
- **n8n (self-hosted) или Make.com** — автоматизация:
  - Ежедневная проверка Notion DB через API
  - Создание напоминаний в Apple Reminders / Google Calendar
  - Отправка уведомлений в Telegram / email
- **Apple Reminders/Calendar** — нативные push-уведомления на iPhone и Mac

**Плюсы:**
- Красивый UI через Notion
- Notion работает на всех платформах
- Нативные уведомления через Apple экосистему
- Notion API хорошо документирован
- Семейный доступ через Notion Shared Workspace

**Минусы:**
- Зависимость от нескольких сервисов
- Make.com платный (бесплатный tier ограничен)
- n8n требует self-hosting
- Сложность настройки интеграций

**Сложность реализации:** ★★★☆☆ (Средняя)
